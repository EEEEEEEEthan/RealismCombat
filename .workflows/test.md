# debug

利用mcp工具运行游戏

如果有指定测试内容，你需要对指定的测试内容进行测试

否则，你根据上下文推断出需要测试的内容

如果都没有，你进行游戏常规流程的测试

## 战斗相关改动必测项

修改战斗流程（例如伤害结算、音效、动画）时，请按照以下步骤进行自测，确保实际战斗中可以触发对应效果：

1. 使用`system_launch_program`启动游戏客户端，等待命令返回主菜单选项。
2. 通过`game_select_option`选择开始新的冒险，按照剧情提示推进直至进入第一场战斗。
3. 在战斗中执行一次普通攻击，确认目标受到伤害时能够听到预期的音效或看到对应表现。
4. 如需重复验证，可继续攻击多次，观察音效是否持续触发且无异常。
5. 测试结束后使用`game_select_option`选择退出游戏，并调用`system_shutdown`释放客户端。

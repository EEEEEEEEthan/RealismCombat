# scan

扫描代码，补充缺失的文档到 `.documents` 文档系统

1. 扫描 `Scripts/` 目录下的所有 C# 文件（`.cs`）
   - 识别重要的类、接口、结构体，特别是：
     - 核心系统类（如 `Combat`、`Game`、`Character` 等）
     - AutoLoad 单例（如 `DialogueManager`、`AudioManager`、`GameServer` 等）
     - 重要的数据结构（如 `BodyPart`、`Item`、`MenuOption` 等）
     - 关键的工具类和扩展方法
   - 分析代码结构，识别功能模块和系统边界

2. 检查 `.documents/` 目录下的现有文档：
   - 查看 `index.md` 了解文档组织结构
   - 检查是否有对应的文档文件记录这些系统
   - 对比代码实现与文档内容，识别缺失或不一致的部分

3. 补充缺失的文档：
   - 对于未文档化的系统，创建新的文档文件或补充到现有文档
   - 文档应包含：
     - 系统的设计原则和架构说明
     - 核心类的功能和使用方式
     - 关键方法和属性的说明
     - 系统间的交互关系
     - 使用示例和注意事项
   - 参考现有文档的格式和风格（如 `combat-system.md`、`ui-system.md`）

4. 更新文档索引：
   - 如果创建了新文档，需要在 `index.md` 中添加对应的链接
   - 确保文档分类合理（开发规范、核心系统、运行时流程、工具与服务等）

5. 文档格式要求：
   - 使用中文编写
   - 使用 Markdown 格式
   - 保持与现有文档风格一致
   - 重点说明系统的用途、设计思路和关键实现细节

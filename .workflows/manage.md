# manage

任务管理和代码审查工作流

## 工作流程

1. **任务分解**
   - 将用户需求拆解为非常细粒度的编码任务
   - 每个任务应该是独立、可执行的单元
   - 明确每个任务的目标和验收标准

2. **任务执行**
   - 对每个编码任务，使用 `qwen -p -y "编码任务"` 让千问执行
   - 等待千问完成编码任务

3. **代码审查**
   - 在千问执行完编码任务后，进行代码审查
   - 检查代码质量、逻辑正确性、是否符合编码规范

4. **处理审查结果**
   - **重大问题**：如果发现重大问题（如逻辑错误、架构问题、安全隐患等），使用 `git clean -df` 清理工作区，然后重新分配任务给千问
   - **细小问题**：如果只有可接受的细小问题（如格式、命名、注释等），可以按照 [commit.md](/.workflows/commit.md) 的流程先提交，之后再优化

5. **任务迭代**
   - 继续下一个编码任务，重复步骤2-4
   - 直到所有任务完成

## 注意事项

- 任务分解要足够细致，确保每个任务都有明确的边界
- 审查时要关注代码的功能正确性和代码质量
- 重大问题需要立即处理，避免问题累积
- 细小问题可以记录并在后续优化中处理
